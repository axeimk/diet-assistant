# プロフィールの形式はJSON Schemaを正本にし、検証は自作する

- ステータス: 承認済み
- 日付: 2026-07-21

`config/profile.json`の形式は`src/diet_assistant/profile.schema.json`（JSON Schema draft
2020-12）を正本とし、`validate_profile`はこのスキーマを読んで検証する。エディタの補完・警告
（`.vscode/settings.json`の`json.schemas`）とCLIの検証が同じ定義を参照するため、項目の追加は
スキーマの修正だけで済む。

## 検討した代替案

- **`jsonschema`パッケージを使う**: 実行時の外部依存が0個から1個になる。このツールは標準
  ライブラリだけで動きネットワークも不要という性質を持つため、10項目程度のフラットな
  オブジェクトを検証するためだけに崩す価値はないと判断した。使う機能は`type`・`enum`・
  `minimum`・`maximum`・`format: date`・`additionalProperties`だけで、自作しても小さい。
- **スキーマはエディタ支援専用にし、検証は手書きのまま**: 実装コストは最小だが、ルールが
  スキーマと`config.py`の2か所に分かれてずれる。

## 結果

自作バリデータはスキーマ全体ではなく上記のキーワードのみを解釈する。スキーマに`$ref`や
`oneOf`などを書いても**黙って無視される**ため、プロフィールの形式はこれらを使わない範囲に
とどめる。スキーマはパッケージデータとして同梱する（`pyproject.toml`の`package-data`）。
`--root`で別のデータルートを指定してもスキーマはパッケージから読むため影響を受けない。
