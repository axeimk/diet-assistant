# ルーティーンは時刻でなく順序で持ち、照合はエージェントが行う

- ステータス: 承認済み
- 日付: 2026-07-23

`config/profile.json` の `routine` は1日のステップ（体重測定・各食事・レポート等）を
実施順に並べた配列で、時刻を持たない。記録の依頼を受けたエージェントが当日の記録と
照合し、先行ステップの抜けを一言確認する。CLIのコードは `routine` を参照しない。

## 検討した代替案

- **時刻ベースのスケジュール（「12時に昼食」）**: 生活の揺らぎで常にずれ、
  誤検知の通知が続くと確認自体が無視されるようになる。順序は揺らぎに強い。
- **CLIコードでの照合・警告**: 「抜けをいつ・どう聞くか」は会話の文脈
  （初回報告か、既に断られたか）に依存する判断で、コードに置くと硬直する。
  スキーマに項目だけ定義しコードから読まない前例は `advice_preference` にもある。

## 結果

ステップ名は `routine` のenum（`weight`・`breakfast`・`lunch`・`snack`・`dinner`・
`exercise`・`report`）に限る。`snack` だけは食べない日があるのが普通の
任意ステップで、抜けていても確認しない（順序上の位置づけにだけ使う）。
`exercise` は任意にしない — ルーティーンに入れた運動は確認してほしいという利用者の意向。
照合手順はrecord-meal / write-reportスキルとAGENTS.mdの「定型作業」節に置く。
